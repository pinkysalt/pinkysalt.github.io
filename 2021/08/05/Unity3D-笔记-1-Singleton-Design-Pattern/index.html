Singleton Design Pattern

概念

Singleton Design Pattern（单一实例模式）是一种游戏设计模式。

  • 全局都可以访问该class，且可直接访问，不需要GetComponent访问
  • 该class的instance应该只存在一个。
  • 比如游戏中的Manager class：Game Manager，Item Manager，UI Manager 等

Lazy Instantiation

有了Singleton，也可以先不创建游戏对象、挂载脚本

这样，即使不存在挂载 UI Manager 脚本的物体，也可以在player中直接调用，运行以后会自动创建游戏对象 UI Manager，停止运行后该游戏对象会被删除。

优点
  • global access conmunication
  • lazily instantiate an object
缺点

需要提前在inspector中拖拽prefab赋值时，由于没有游戏对象，无法赋值
